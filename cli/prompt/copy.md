위 내용을 참고하여 해당 개념/주제에 대한 완벽한 마크다운 문서를 같은 경로에 생성해줘.

작성 규칙:

1. 파일명

    - 주제의 핵심이 드러나는 이름으로 작성
    - 대소문자와 언더스코어(\_) 혼용
    - 예: Feature_Sliced_Design.md, React_Server_Components.md

2. 작성 스타일

    - 이모지 사용 금지
    - 전문적이고 간결한 기술 문서
    - 핵심 용어는 영어, 설명은 한글
    - 일상 예시 금지, 기술적 설명만
    - 중복 표현 금지
    - frontmatter(---, created 등의 메타데이터) 절대 사용 금지

3. 문서 구조

# {파일명과 동일한 제목}

## 개요

개념을 2-3문장으로 명확하게 정의

## 주요 내용

핵심 내용을 섹션별로 구분하여 상세히 설명
필요한 경우 코드 예제를 최소한으로 포함

## 요약

핵심 포인트를 간결하게 정리

## 참고

관련 공식 문서 및 레퍼런스 링크

    - 주석은 필요한 부분만
